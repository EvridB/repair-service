# Веб-сервис «Заявки в ремонтную службу»

Тестовое задание на позицию Вайб-кодер в компанию «База Бизнеса». 
Система для управления ремонтными заявками с ролевой моделью (Диспетчер/Мастер) и защитой от параллельных запросов.

## 👥 Тестовые пользователи (Seeders)

Для входа в систему используйте следующие данные (пароль для всех: `password`):


| Роль      | Email              | Имя              |
|-----------|--------------------|------------------|
| Диспетчер | `admin@test.com`   | Диспетчер Иван   |
| Мастер 1  | `master1@test.com` | Мастер Алексей   |
| Мастер 2  | `master2@test.com` | Мастер Сергей    |

*После запуска Docker-контейнера база уже будет заполнена этими пользователями и 15-ю случайными заявками.*

## 🏎 Проверка Race Condition (Защита от гонки)

Для реализации защиты используется механизм БД `lockForUpdate()`.  
Чтобы проверить, что два мастера не могут взять одну заявку одновременно:

1. Откройте два окна терминала.
2. В каждом подготовьте команду (но не нажимайте Enter):
   ```bash
   curl -X POST http://localhost:8000/api/requests/16/take \
   -H "Content-Type: application/json" \
   -d '{"user_id": 2}'
