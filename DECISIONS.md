# Архитектурные решения (DECISIONS.md) — v2.0

В данном проекте реализованы следующие ключевые решения для обеспечения надежности и скорости разработки:

### 1. Выбор стека (Laravel 11 + PostgreSQL)
Laravel выбран как наиболее эффективный фреймворк для быстрой реализации бизнес-логики с помощью AI-ассистентов. PostgreSQL использован из-за его продвинутой поддержки транзакций и блокировок, необходимых для решения задачи Race Condition.

### 2. Защита от гонки данных (Race Condition)
Для метода `takeToWork` реализована стратегия **Pessimistic Locking** (`lockForUpdate()`) внутри БД-транзакции. 
- **Почему:** Обычная проверка статуса через `if` на уровне PHP не защищает от одновременных запросов. Блокировка на уровне строки БД гарантирует, что только один поток успешно обновит статус, а остальные получат корректный ответ `409 Conflict`.

### 3. Контейнеризация (Docker-first подход)
Проект полностью автономен. 
- **Решение:** Весь процесс сборки (установка Composer, генерация ключей, настройка прав) вынесен в `Dockerfile`. 
- **Цель:** Исключить ошибку "у меня не запускается", обеспечив старт одной командой `docker compose up`.

### 4. Наполнение базы (Seeding Strategy)
Для удобства тестирования реализован `DatabaseSeeder` совместно с `Model Factories`.
- **Решение:** При запуске сидов создаются не только роли, но и 15+ заявок с разными статусами. Это позволяет сразу проверить работу фильтров диспетчера без ручного ввода данных.

### 5. Применение паттерна REST для смены статусов
Для операций «Отмена» и «Завершение» использован метод **PATCH**. 
- **Почему:** Согласно стандартам REST, частичное обновление ресурса (смена одного поля статуса) должно происходить именно через PATCH, что демонстрирует понимание протоколов взаимодействия.

### 6. Использование AI (Вайб-кодинг)
Разработка велась с активным применением **GitHub Copilot**. 
- **Подход:** AI использовался для генерации бойлерплейта и CRUD-логики, в то время как за мной оставался архитектурный надзор, проектирование связей в БД и отладка критических узлов (Docker, Lock логика).
